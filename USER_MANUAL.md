# Vectra Control Outreach User Manual

Vectra Control Outreach is a safety-first LinkedIn outreach dashboard. It helps you manage LinkedIn accounts, proxies, saved sessions, leads, scraping campaigns, connection requests, follow-up messages, Sales Navigator InMail workflows, checkpoints, activity logs, and queue jobs.

Use Vectra only with LinkedIn accounts you control and keep limits conservative. The app includes guardrails, but it does not remove LinkedIn platform risk.

## 1. Core Concepts

### Accounts

An account is one LinkedIn identity. Each account has its own:

- LinkedIn email
- timezone and active hours
- residential proxy
- saved LinkedIn session cookies
- warm-up phase
- daily action caps
- checkpoint status
- Sales Navigator / InMail settings

Automation will not run correctly until the account has a stable proxy and a saved LinkedIn session.

### Proxies

Each LinkedIn account should use one stable residential proxy in the same country or city the account normally logs in from. Vectra blocks browser jobs when an account has no proxy.

Recommended proxy type: sticky residential or static residential / ISP.

Avoid datacenter, rotating, inactive, IPv6, or unstable proxies.

### Campaigns

Campaigns define what Vectra should do with a set of leads. Supported campaign types:

- `CONNECT`: send LinkedIn connection requests.
- `MESSAGE`: send follow-up messages to already-connected leads.
- `INMAIL`: send Sales Navigator InMail to Sales Navigator leads.
- `SCRAPE`: scrape LinkedIn or Sales Navigator search/profile data without sending messages.
- `CONTENT_SIGNAL`: find LinkedIn authors who recently posted about a keyword/topic.

### Leads

Leads are LinkedIn profiles stored in Vectra. They can be added manually, imported by CSV, collected from LinkedIn search URLs, collected from Sales Navigator URLs, or generated by Content Signal.

### Guardrails

Vectra enforces daily caps, warm-up phases, active-hour windows, proxy requirements, checkpoint pauses, and queue-level safety checks. Campaign daily limits do not override account-level safety caps.

## 2. First-Time Setup

### Step 1: Sign In

Open the dashboard and sign up or log in.

Production dashboard:

```text
https://linkedin-automation-dashboard-6gy0.onrender.com
```

Production API:

```text
https://api-production-ce2ca.up.railway.app
```

### Step 2: Add a Proxy

Go to `Proxies`.

You can either:

- click `Add Proxy` and enter proxy details manually, or
- use `Load Proxy-Cheap` if Proxy-Cheap credentials are configured.

Manual proxy fields:

- Host
- Port
- Country
- City, optional
- Username
- Username template, optional
- Password
- Rotation mode

For sticky proxy usernames, the username template must include:

```text
{{sessionId}}
```

After adding a proxy, click `Health Check`. Use only proxies that return healthy status and stable exit IPs.

### Step 3: Add a LinkedIn Account

Go to `Accounts`, then click `Add Account`.

Enter:

- LinkedIn email
- timezone
- proxy
- Sales Navigator enabled, only if the LinkedIn account actually has Sales Navigator
- monthly InMail limit, if Sales Navigator/InMail is enabled

If the selected proxy location does not match the timezone, Vectra warns you. Use the location this account normally logs in from.

### Step 4: Connect LinkedIn Session

On the account card, click `Connect LinkedIn` or `Refresh LinkedIn`.

Follow the on-screen steps:

1. Install Cookie-Editor.
2. Log in to the correct LinkedIn account in your browser.
3. Export cookies from Cookie-Editor.
4. Paste the exported cookie JSON into Vectra.
5. Check the consent box.
6. Click `Save session`.

The cookie JSON must be a JSON array. Vectra encrypts and reuses this saved session for campaign browser runs.

## 3. Account Operations

On the `Accounts` page, each account card shows health, status, warm-up phase, proxy, timezone, saved session, and open checkpoints.

### Pause Automation

Use `Pause automation` to temporarily stop work for an account.

### Resume Automation

Use `Resume automation` after a pause. If the account is restricted, resolve the restriction first.

### Advance Warm-Up

Use `Advance warm-up` only after the account has stayed healthy at the current phase. Warm-up affects safe sending volume.

### De-Advance Warm-Up

Use `De-advance warm-up` to reduce account volume after errors, warnings, or risky behavior.

### Edit Daily Limits

Use `Edit daily limits` to tune caps for:

- connections
- messages
- InMails
- profile views
- search pages

Vectra still enforces hard server ceilings.

## 4. Lead Management

Go to `Leads`.

### Add One Lead

Use `Add Single Lead`.

Required:

- LinkedIn URL

Optional:

- first name
- last name
- company
- title
- campaign assignment

### Import Leads by CSV

Use `Bulk CSV Import`.

Required column:

```text
linkedinUrl
```

Accepted LinkedIn URL header aliases include:

```text
url, linkedinUrl, linkedin_url, linkedin url, profile url, profileUrl
```

Optional columns:

```text
firstName,lastName,company,title
```

Example:

```csv
linkedinUrl,firstName,lastName,company,title
https://www.linkedin.com/in/example,Alice,Smith,Acme,CEO
```

For Sales Navigator InMail workflows, Sales Navigator lead URLs are supported:

```csv
linkedinUrl,firstName,lastName,company,title
https://www.linkedin.com/sales/lead/ACwAA...,Alice,Smith,Acme,CEO
```

You can attach imported leads to a campaign during import.

## 5. Campaign Workflow

Go to `Campaigns`, then click `Create Campaign`.

Choose:

- campaign name
- LinkedIn account
- campaign type
- daily limit
- target timezone, optional
- connection note, for connection campaigns

After creating the campaign, open its campaign detail page to add leads, templates, search URLs, or Content Signal settings.

### Campaign Status

Campaigns can be started, paused, duplicated, edited, or deleted from the campaign screens.

Starting a campaign queues work. The worker then runs jobs according to account caps, active hours, campaign type, and lead eligibility.

## 6. Connection Campaigns

Use `CONNECT` when you want Vectra to send LinkedIn connection requests.

Recommended lead type: people you are not yet connected with.

Connection notes support variables:

```text
{{firstName}}
{{lastName}}
{{company}}
{{title}}
```

LinkedIn connection notes are limited to 300 characters. Leave the note blank if you want to send without a note.

Vectra skips or warns on leads that are already connected.

## 7. Message Campaigns

Use `MESSAGE` when you want to send messages to existing first-degree connections.

Important: LinkedIn direct messages only work for connected leads. If a lead is not connected, Vectra marks that mismatch and the job should not send.

On the campaign detail page, build the message sequence:

- add message steps
- edit message content
- set delays
- reorder steps by dragging

Each step runs after its delay has passed since the previous step.

## 8. Sales Navigator InMail Campaigns

Use `INMAIL` only when the LinkedIn account has Sales Navigator and available InMail access.

Account requirements:

- `Sales Navigator enabled` must be turned on in the account settings.
- The real LinkedIn account must have an active Sales Navigator subscription.
- The monthly InMail limit should match the account's real available InMail credits.

Lead recommendations:

- Sales Navigator lead URLs, such as `/sales/lead/...`
- imported Sales Navigator lead lists

InMail is separate from connection requests and LinkedIn direct messages. Vectra does not automatically turn a normal message campaign into InMail. Use an `INMAIL` campaign and configure the InMail template.

If the account does not have Sales Navigator, LinkedIn redirects to `/sales/login`, and Sales Navigator scraping/InMail will not work.

## 9. Scrape Campaigns

Use `SCRAPE` when you want to collect profile data without sending outreach.

You can add:

- individual LinkedIn profile URLs
- LinkedIn people search URLs
- Sales Navigator people search/list URLs, only for Sales Navigator accounts

On the campaign detail page, click `Search URL`.

Choose the source:

- `LinkedIn people search`
- `Sales Navigator search/list`

For normal LinkedIn search, use URLs like:

```text
https://www.linkedin.com/search/results/people/?keywords=doctor
```

For Sales Navigator search, use URLs like:

```text
https://www.linkedin.com/sales/search/people?query=...
```

Vectra validates the source. A `/sales/...` URL must be submitted as Sales Navigator, and Sales Navigator scraping is blocked unless the account has Sales Navigator enabled.

You can also use the built-in URL builder for normal LinkedIn people search by entering keywords, title, location, and connection degree.

## 10. Content Signal Campaigns

Use `CONTENT_SIGNAL` when you want to find people who recently posted about a topic.

On the campaign detail page, configure:

- keyword
- date range
- max leads
- title filter, optional
- company filter, optional
- connection note template, optional

Run the Content Signal scraper. Vectra stores each matching author as a lead and saves the post excerpt/context next to the lead.

Use this when you want connection notes that reference what someone recently wrote.

## 11. Starting a Campaign

Before starting:

1. Confirm the account is `ACTIVE`.
2. Confirm the account has a healthy proxy.
3. Confirm the LinkedIn session is saved.
4. Confirm there are no open checkpoints.
5. Confirm the campaign has leads or a configured source.
6. Confirm the campaign type matches the lead type.
7. Keep daily limits low for new accounts or new workflows.

Then click `Start` on the campaign.

After starting, monitor:

- campaign lead statuses
- `Jobs`
- `Activity Log`
- `Rate Limits`
- `Checkpoints`

## 12. Monitoring

### Dashboard

Use the main dashboard for a quick overview of account health, open checkpoints, recent activity, and operational status.

### Activity Log

Go to `Activity Log` to see every automated action.

You can filter by:

- account
- action type

Supported action types include:

- connect
- message
- scrape
- search_scrape
- content_signal
- withdraw
- checkpoint_detected

You can export up to 10,000 rows as CSV.

### Jobs

Go to `Jobs` to inspect queue state.

Filter by state:

- active
- waiting
- delayed
- completed
- failed

Filter by queue:

- connect
- message
- scrape
- searchScrape
- withdraw
- sequenceDispatch
- contentSignal
- anomalyCheck
- syncStatus

Use failed job details to understand browser errors, validation failures, proxy failures, or LinkedIn session issues. Use `Clear failed jobs` only after you have captured any error detail you need.

### Rate Limits

Go to `Rate Limits` to see daily cap usage by account.

Colors:

- red: 80% or more used
- amber: 60% or more used
- normal: below 60%

Caps reset at midnight in each account's local timezone.

## 13. Checkpoints

Vectra pauses accounts when LinkedIn security prompts or checkpoints are detected.

If a checkpoint appears:

1. Go to `Checkpoints`.
2. Log in to LinkedIn manually for the affected account.
3. Complete any CAPTCHA, identity verification, or security prompt.
4. Confirm the account is accessible and not restricted.
5. Click `Mark Resolved`.
6. Return to `Accounts` and resume automation if needed.

Accounts with repeated checkpoints are automatically treated more conservatively.

## 14. Settings and Alerts

Go to `Settings` to configure alert delivery.

Alerts can be sent for:

- checkpoint detection
- account pauses caused by anomalies
- proxy health degradation

Webhook alerts are compatible with Slack, Discord, and generic webhook receivers.

Use `Test alert` after saving a webhook URL.

## 15. Sales Navigator Rules

Sales Navigator support is explicit in Vectra.

Use Sales Navigator when:

- the LinkedIn account has an active Sales Navigator subscription
- you are importing `/sales/...` URLs
- you are scraping Sales Navigator people search/list pages
- you are running InMail campaigns

Do not enable Sales Navigator for accounts without a subscription. LinkedIn will redirect those sessions to `/sales/login`, and Vectra will not be able to scrape Sales Navigator leads or send InMail.

InMail credits are separate from:

- connection request limits
- direct message limits
- normal LinkedIn search scraping

Set the monthly InMail limit to the number of credits you want Vectra allowed to use.

## 16. Common Problems

### Sales Navigator scrape returns 0 leads

Likely causes:

- the account does not have Sales Navigator
- the account was redirected to `/sales/login`
- the URL source was set incorrectly
- the saved LinkedIn session expired
- LinkedIn changed the page layout

Fix:

1. Confirm the real LinkedIn account has Sales Navigator.
2. Refresh the saved LinkedIn session cookies.
3. Enable `Sales Navigator enabled` on the account.
4. Submit the URL with source `Sales Navigator search/list`.

### Jobs are not running

Check:

- account status is `ACTIVE`
- proxy is assigned and healthy
- session cookies are saved
- no open checkpoints
- daily caps are not exhausted
- campaign is started
- jobs are not delayed because of active-hour scheduling

### Message campaign skips leads

Most likely the lead is not connected. Direct LinkedIn messages require first-degree connection status.

Use a `CONNECT` campaign first, then move accepted leads into a `MESSAGE` campaign.

### InMail campaign fails or does nothing

Check:

- account has real Sales Navigator access
- Sales Navigator is enabled in Vectra for the account
- monthly InMail cap is not exhausted
- lead URLs are Sales Navigator lead URLs or profiles with InMail available
- saved session is still valid

### Proxy warning appears

The proxy country/city does not match the selected account timezone. Use the location the account normally uses.

### Checkpoint detected

Stop automation for that account, resolve LinkedIn manually, then mark the checkpoint resolved in Vectra.

## 17. Recommended Operating Routine

Daily:

1. Check `Dashboard`.
2. Review `Checkpoints`.
3. Review `Rate Limits`.
4. Check failed jobs.
5. Inspect recent activity.

Before launching a new campaign:

1. Start with a low daily limit.
2. Test with a small lead list.
3. Watch the first few jobs.
4. Confirm activity looks correct.
5. Scale gradually only if the account remains healthy.

For new accounts:

- keep warm-up low
- avoid high connection limits
- use a stable residential proxy
- avoid changing proxy location frequently
- refresh cookies if LinkedIn challenges the account

## 18. Quick Start Checklist

Use this checklist for a fresh operator:

1. Add a healthy residential proxy.
2. Add the LinkedIn account.
3. Match account timezone and proxy location.
4. Save LinkedIn session cookies.
5. Keep warm-up conservative.
6. Add or import leads.
7. Create the correct campaign type.
8. Add templates or search URLs.
9. Start with a small daily limit.
10. Monitor jobs, activity, limits, and checkpoints.
